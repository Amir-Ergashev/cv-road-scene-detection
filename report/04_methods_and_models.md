# 3.5 Методы и модели

В рамках проекта реализовано и обучено пять современных моделей детекции
объектов, охватывающих основные архитектурные парадигмы, рассмотренные в
литературном обзоре (раздел 3.2): одностадийные детекторы (YOLOv8, SSD),
двухстадийный детектор (Faster R-CNN), детектор с многомасштабным слиянием
признаков (EfficientDet) и трансформерный детектор (DETR).

## 3.5.1 YOLOv8 (You Only Look Once v8)

**Архитектура.** Однопроходный (one-stage) детектор: единая свёрточная сеть
предсказывает bounding box'ы и классы напрямую по всему изображению.
Использована облегчённая версия **YOLOv8n (nano)** — наименьшая по числу
параметров модель семейства, оптимальная при ограниченных вычислительных
ресурсах.

**Гиперпараметры:**

| Параметр | Значение |
|---|---|
| Backbone/архитектура | YOLOv8n (предобученная на COCO) |
| Размер изображения | 640×640 |
| Batch size | 16 |
| Эпохи | 50 (early stopping, patience=10) |
| Оптимизатор | AdamW (автоподбор lr0, momentum) |
| Аугментации | mosaic, flip, color jitter (встроенные в ultralytics) |
| Устройство | GPU (NVIDIA RTX 4070) |

**Обоснование выбора:** YOLOv8 выбран как baseline-модель проекта благодаря
быстрому обучению и инференсу, готовой библиотечной реализации (ultralytics)
и широкому применению в задачах реального времени, включённого в дорожную
тематику проекта.

## 3.5.2 Faster R-CNN

**Архитектура.** Двухстадийный детектор: Region Proposal Network (RPN)
генерирует предложения регионов, после чего отдельная головка классифицирует
и уточняет координаты bounding box'ов. Backbone — ResNet-50 с Feature
Pyramid Network (FPN).

**Гиперпараметры:**

| Параметр | Значение |
|---|---|
| Backbone | ResNet-50-FPN (предобучен на COCO) |
| Размер изображения | 480–640 px (min_size/max_size, уменьшено с дефолтных 800/1333 для ускорения) |
| Batch size | 16 |
| Эпохи | 15 |
| Оптимизатор | SGD (lr=0.005, momentum=0.9, weight_decay=0.0005) |
| LR schedule | StepLR (step_size=5, gamma=0.1) |
| Mixed precision | включен (AMP) |
| Устройство | GPU (NVIDIA RTX 4070) |

**Обоснование выбора:** Faster R-CNN — эталонная двухстадийная архитектура,
востребованная в задачах, где приоритетна точность, а не скорость
инференса; включена в проект как представитель этого подхода для
сравнения с одностадийными альтернативами.

## 3.5.3 SSD (Single Shot MultiBox Detector)

**Архитектура.** Одностадийный детектор с многомасштабными картами
признаков для детекции объектов разных размеров за один проход сети.
Backbone — VGG16.

**Гиперпараметры:**

| Параметр | Значение |
|---|---|
| Backbone | VGG16 (предобучен на ImageNet) |
| Размер изображения | 300×300 (фиксированный, архитектурное ограничение SSD300) |
| Batch size | 16 |
| Эпохи | 15 |
| Оптимизатор | SGD (lr=0.0015, momentum=0.9, weight_decay=0.0005) |
| LR schedule | Linear warmup (200 шагов) + StepLR (step_size=7, gamma=0.1) |
| Постобработка | score_thresh=0.2, detections_per_img=100 |
| Устройство | GPU (NVIDIA RTX 4070) |

**Обоснование выбора:** SSD представляет альтернативный одностадийный
подход к YOLO с другим механизмом работы с разномасштабными объектами;
включён для сравнения эффективности устаревающего backbone (VGG16)
относительно современных архитектур.

## 3.5.4 EfficientDet

**Архитектура.** Использует EfficientNet в качестве backbone и
взвешенную двунаправленную пирамиду признаков (BiFPN) для эффективного
многомасштабного слияния признаков. Применена самая лёгкая версия
семейства — **EfficientDet-D0**.

**Гиперпараметры:**

| Параметр | Значение |
|---|---|
| Backbone | EfficientNet-B0 (предобучен на ImageNet) |
| Размер изображения | 512×512 |
| Batch size | 8 |
| Эпохи | 15 |
| Оптимизатор | SGD (lr=0.001, momentum=0.9, weight_decay=0.0005) |
| LR schedule | Linear warmup (200 шагов) + StepLR (step_size=7, gamma=0.1) |
| Реализация | библиотека effdet (Ross Wightman) + timm |
| Устройство | GPU (NVIDIA RTX 4070) |

**Обоснование выбора:** EfficientDet заявлен авторами как архитектура с
оптимальным балансом точности и эффективности (число параметров/FLOPs);
включён в проект для проверки этого свойства на собственном датасете.

## 3.5.5 DETR (Detection Transformer)

**Архитектура.** Трансформерный детектор: CNN-backbone (ResNet-50)
извлекает признаки, энкодер-декодер трансформер с фиксированным набором
из 100 object queries напрямую предсказывает множество объектов без
анкорных боксов и NMS.

**Гиперпараметры:**

| Параметр | Значение |
|---|---|
| Backbone | ResNet-50 (предобучен на COCO, чекпойнт facebook/detr-resnet-50) |
| Размер изображения | 480–640 px (shortest/longest edge, уменьшено с дефолтных 800/1333) |
| Batch size | 8 |
| Эпохи | 8 |
| Оптимизатор | AdamW (lr=1e-5, weight_decay=1e-4) |
| LR schedule | StepLR (step_size=5, gamma=0.1) |
| Реализация | библиотека transformers (HuggingFace) |
| Устройство | GPU (NVIDIA RTX 4070) |

**Обоснование выбора:** DETR представляет принципиально иной,
трансформерный подход к детекции без вручную спроектированных
компонентов (anchors, NMS); включён в проект как представитель этого
современного направления, несмотря на известное ограничение —
необходимость большого числа эпох обучения для полной сходимости (см.
раздел 3.2 отчёта).

## Сводная таблица моделей

| Модель | Тип | Backbone | Разрешение | Параметры (млн) |
|---|---|---|---|---|
| YOLOv8n | one-stage | CSPDarknet (custom) | 640×640 | ~3.0 |
| Faster R-CNN | two-stage | ResNet-50-FPN | 480-640 | ~41.5 |
| SSD300 | one-stage | VGG16 | 300×300 | ~24.4 |
| EfficientDet-D0 | one-stage (BiFPN) | EfficientNet-B0 | 512×512 | ~3.9 |
| DETR | transformer | ResNet-50 | 480-640 | ~41.5 |

Все модели обучены с использованием предобученных весов (transfer
learning) на полном датасете COCO (80/91 классов) с последующей заменой
выходной головы под 6 классов целевой предметной области (раздел 3.1).
Обучение всех моделей проводилось на GPU NVIDIA RTX 4070 Laptop
(8 ГБ видеопамяти) с использованием библиотек PyTorch, torchvision,
ultralytics, effdet и transformers.
