# 3.9 Заключение

## 3.9.1 Основные выводы

В рамках учебной практики реализован проект полного цикла в области
компьютерного зрения, посвящённый детектированию и классификации
объектов на изображениях дорожной обстановки (интеллектуальные
транспортные системы). Выполнены все этапы, предусмотренные методическим
указанием: научный литературный обзор (17 источников), формализация
постановки задачи, подготовка и анализ датасета (подмножество COCO,
6 классов, 2972 изображения), реализация и обучение пяти современных
моделей детекции объектов, проведение сравнительного экспериментального
анализа.

**Основные выводы по итогам сравнения моделей:**

1. **Наилучший результат по интегральной метрике F1-score** показали
   YOLOv8n (0.405) и Faster R-CNN (0.404) — практически идентичный
   результат, достигнутый принципиально разными архитектурными
   подходами (одностадийная регрессия против двухстадийной генерации
   предложений с уточнением).
2. **Наилучший результат по mAP@0.5** показал Faster R-CNN (0.514),
   подтверждая известное из литературы [4] преимущество двухстадийных
   архитектур в задачах, где приоритетна точность детекции, а не
   скорость инференса.
3. **Наилучший баланс скорости обучения и точности** показал YOLOv8n —
   обучение проходило почти в 10 раз быстрее, чем Faster R-CNN, при
   сопоставимом итоговом F1-score, что делает его предпочтительным
   выбором для прикладных задач, требующих частого переобучения модели
   или работы в условиях ограниченных вычислительных ресурсов.
4. **SSD300 показал наименьшую точность** среди всех пяти моделей,
   что согласуется с известным архитектурным ограничением устаревшего
   backbone VGG16 и подтверждается независимыми источниками из
   литературного обзора.
5. **Дисбаланс классов в датасете** (76% объектов класса "person")
   оказал прямое и измеримое влияние на качество детекции редких
   классов независимо от используемой архитектуры — это формирует
   отдельное практическое наблюдение проекта, выходящее за рамки
   простого сравнения моделей.

## 3.9.2 Рекомендация по выбору модели

Для прикладной задачи детекции объектов дорожной обстановки в условиях,
аналогичных рассмотренным в проекте (ограниченный размер обучающей
выборки, умеренные вычислительные ресурсы), рекомендуется **YOLOv8** —
как модель, обеспечивающая лучший баланс между точностью, скоростью
обучения и инференса, что особенно важно для систем, работающих в
реальном времени (анализ дорожного движения, системы помощи водителю).
Если приоритетна максимальная точность детекции, а ограничения по
времени инференса отсутствуют — обоснованным выбором является
**Faster R-CNN**.

## 3.9.3 Направления дальнейших исследований

1. **Увеличение обучающей выборки.** Использование полного датасета
   COCO для выбранных классов (а не подмножества из 5000 изображений
   val2017) потенциально улучшит результаты всех моделей, в особенности
   SSD и EfficientDet с большим числом обучаемых параметров.
2. **Систематический подбор гиперпараметров.** Применение формальных
   методов оптимизации (grid search, Bayesian optimization, Optuna) для
   подбора learning rate, batch size и других параметров вместо
   эвристических значений из документации.
3. **Увеличение числа эпох для DETR.** Учитывая выявленную в обзоре
   литературы потребность DETR в большом числе эпох для полной
   сходимости, эксперимент с 50–100 эпохами (при наличии вычислительных
   ресурсов) может существенно изменить итоговую оценку этой модели.
4. **Учёт дисбаланса классов.** Применение методов балансировки
   (weighted loss, oversampling редких классов, focal loss) для
   снижения выявленного разрыва в качестве детекции между частыми и
   редкими классами.
5. **Расширение набора метрик.** Дополнительный анализ ошибок по
   размеру объектов (мелкие/средние/крупные, как в стандартной
   методологии COCO eval) для более детального понимания слабых сторон
   каждой модели — особенно с учётом обсуждаемой в литературе проблемы
   детекции мелких объектов (раздел 3.2, источник [17]).
6. **Тестирование в неблагоприятных условиях.** Оценка устойчивости
   моделей к погодным условиям и низкой освещённости, что является
   значимым открытым вопросом для систем восприятия в реальных
   интеллектуальных транспортных системах (раздел 3.2, источники
   [12, 13]).
